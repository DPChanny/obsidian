## 1. 역할
작업 디렉토리의 변경 사항을 스테이징 영역(Index)에 추가하여 다음 커밋에 포함될 준비를 함.

## 2. 주요 사용 사례

### 2.1. 특정 대상 스테이징
- `git add <file>`: 특정 파일만 스테이징함.
- `git add <dir>`: 디렉토리 내 모든 변경 사항을 스테이징함.

### 2.2. 일괄 스테이징
- `git add .`: 현재 경로의 모든 새 파일 및 수정 사항을 반영함.
- `git add -u`: 이미 추적 중인 파일의 수정/삭제만 반영함.
- `git add -A`: 새 파일, 수정, 삭제를 포함한 모든 변경 사항을 반영함.

### 2.3. 부분 스테이징
- `git add -p`: 변경 사항을 Hunk 단위로 나누어 대화식으로 스테이징함.

## 3. 문법 요약
```bash
# 전체 스테이징
git add -A

# 상태 확인
git status
```
